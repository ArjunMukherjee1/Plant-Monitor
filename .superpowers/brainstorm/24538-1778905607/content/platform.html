Where should the app run?

This shapes the entire tech stack and how you'll access sensor data

A

Web App (browser-based)

Runs in any browser on any device — phone, tablet, laptop. Hosted locally (e.g. on the same machine as Home Assistant) or on the internet. Best for flexibility and easy sharing.

  • Works on any device with a browser
  • Easy to update — just refresh
  • Can be embedded in Home Assistant dashboard
B

Mobile App (iOS / Android)

A native app on your phone. Great for push notifications (e.g. "soil is too dry!") and home screen access. More work to build and distribute.

  • Push notifications for alerts
  • Home screen icon
  • Requires App Store / TestFlight distribution
C

Desktop App (Mac / Windows)

A native desktop app, e.g. built with Electron or Tauri. Lives in your taskbar or dock. Good for always-on monitoring on a dedicated machine.

  • Runs in background, system tray
  • Good for a dedicated monitor/screen
  • Less portable than a web app
